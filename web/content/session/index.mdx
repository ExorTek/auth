# `@exortek/session`

Sealed-cookie session manager for Node.js ‚Äî rotation, revocation, sudo
mode, impersonation, concurrent limits, fingerprint binding, device
labels, session events, Redis distributed revocation, CSRF derivation,
trusted-device cookies, and adapters for Fastify and Express. Built on
`@exortek/crypto`.

## Quick tour

```js
import { createSessionManager } from '@exortek/session';

const sessions = createSessionManager({
  secret: process.env.SESSION_SECRET,
  ttl:     '7d',
  idleTtl: '30m',
});

// Login
const { cookie } = await sessions.issue({
  userId: user.id,
  claims: { roles: ['admin'] },
});
res.setHeader('Set-Cookie', cookie);

// Middleware
const session = await sessions.verify(req);
if (!session) return unauthorized();

// Password-change flow ‚Äî sign out every other device
await sessions.revokeAllExceptCurrent(req);

// Sensitive endpoint ‚Äî sudo mode
if (!(await sessions.requireFreshAuth(req, { maxAgeSeconds: 300 }))) {
  return res.redirect('/reauth');
}

// Logout
const { cookie: bye } = await sessions.revoke(req);
res.setHeader('Set-Cookie', bye);
```

## Features (opt-in via config flags)

- **`anonymous: true`** ‚Äî guest sessions with `userId: null`; promote via `sessions.upgrade()`
- **`concurrentLimit: 3`** ‚Äî max sessions per user; oldest kicked
- **`bindTo: ['ip', 'ua']` + `bindStrictness: 'soft'`** ‚Äî fingerprint binding with mobile-friendly mode
- **`impersonation: true`** + **`impersonationTtl: '15m'`** ‚Äî admin-as-user with a short TTL and full audit trail
- **`deviceLabels: true`** ‚Äî auto-generated `iPhone 14 ¬∑ Chrome`
- **`events: { onIssue, onVerify, onRotate, onRevoke, onDeny, onSuspicious }`** ‚Äî audit callbacks
- **`suspiciousActivity: true`** ‚Äî IP + UA change flagging
- **`touchEvery: '1m'`** ‚Äî cut store write traffic on hot login endpoints
- **`store: redisStore(...)`** ‚Äî Redis session store with pub/sub distributed revocation
